### 질문 1: 직접 구현한 AABB와 Pygame 내장 함수의 차이
> 강의 영상에 나온 `aabb_collision` 조건문과 AI가 작성해준 `colliderect` 조건문은 어떤 차이가 있나요?

**[강의 영상 코드 - 수동 구현]**
```python
def aabb_collision(x1, y1, w1, h1, x2, y2, w2, h2):
    if x1+w1 < x2: return False # A가 B 왼쪽
    if x2+w2 < x1: return False # A가 B 오른쪽
    if y1+h1 < y2: return False # A가 B 위
    if y2+h2 < y1: return False # A가 B 아래
    return True
```

**[Pygame 내장 함수 - 자동 구현]**
```python
elif center_rect.colliderect(moving_rect):
```

**답변:**
* **원리:** 수학적 충돌 판정 원리는 완전히 동일합니다. 강의 코드는 그 원리를 학습하기 위해 파이썬으로 직접 풀어서 구현한 것입니다.
* **속도와 편의성:** 내장 함수(`colliderect`)는 속도와 편의성을 위해 Pygame 내부에 최적화되어 있습니다. 직접 작성한 코드는 파이썬 엔진이 매번 한 줄씩 해석하며 실행하기 때문에 상대적으로 느립니다. 반면, Pygame의 내장 함수는 내부적으로 속도가 훨씬 빠른 C언어로 미리 컴파일되어 작동하므로 연산 속도에서 큰 차이가 납니다.

### 질문 2: 수학 모듈(math)의 활용 차이
> 원형 충돌 감지에서는 `math.sqrt`를 자주 쓰지 않나요? 사각형을 감싸는 외접원을 그릴 때 `math.hypot`를 썼던 이유는 무엇인가요? 그리고 내접원을 그릴 때에는 `math`를 안 쓴 이유가 무엇이죠?

**답변:**
* **원형 충돌:** 정석은 `math.sqrt`를 사용하여 실제 거리를 구하는 것이지만, 루트 연산은 컴퓨터에게 부담이 큽니다. 따라서 게임에서는 연산 최적화를 위해 루트를 빼고 거리의 제곱끼리 비교하는 방식을 씁니다.
* **외접원:** 사각형 중심에서 꼭짓점까지의 대각선 거리를 구해야 하므로 피타고라스의 정리를 활용하는 `math.hypot`이 필요했습니다.
* **내접원:** 단순히 사각형 너비의 절반이 반지름이 되므로, 복잡한 수학 모듈 없이 바로 구할 수 있습니다.

## 각 충돌 감지 방식의 차이점

* **원형 vs AABB**
    AABB는 날카로운 모서리를 기준으로 충돌을 판단하는 반면, 원형은 모서리가 존재하지 않기 때문에 물체의 회전 상태와 상관없이 항상 일정한 충돌 범위를 가집니다.
* **AABB vs OBB**
    AABB는 물체가 회전하여도 충돌 박스는 고정되어 그대로지만, OBB는 물체의 회전을 반영하여 함께 회전합니다. 따라서 OBB는 정밀도 측면에서 원형과 AABB의 장점을 합친 것이라 할 수 있습니다.
